•	DTO는 데이터 전송 객체로서, 데이터의 구조를 정의하고 데이터의 전송을 쉽게 하기 위해 사용됩니다.
•	일반적으로 DTO는 데이터베이스나 외부 시스템과의 데이터 교환을 위해 사용될 수 있습니다.
•	DTO는 순수하게 데이터의 구조와 타입을 정의하는 데 목적이 있으며, 보통 비즈니스 로직이나 상태 관리와는 직접적인 연관이 없습니다.

•	구조화된 데이터: DTO는 데이터 구조를 정의하는 클래스입니다. 예를 들어, 사용자 정보, 제품 정보, 주문 정보 등과 같이 어떤 데이터의 필드와 타입을 명확히 정의합니다.
•	데이터 전송: DTO는 데이터를 전송하는 데 사용될 수 있습니다. 클라이언트와 서버 간의 데이터 교환에서 DTO를 사용하여 데이터를 캡슐화하고, 일관된 데이터 형식을 유지할 수 있습니다.
